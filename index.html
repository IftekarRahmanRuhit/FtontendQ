HTML(Hypertext Markup Language) related interview questions

Question: What is HTML, and why is it important in web development?

Answer: HTML (HyperText Markup Language) is the standard language used to create the structure of web pages. It defines the layout and elements (like text, images, links, etc.) that appear on a webpage. Without HTML, web pages would have no structure for browsers to render.

Question: What are the main parts of an HTML document?

Answer: An HTML document generally includes:

  • < !DOCTYPE html >: Declares the document type as HTML5
  • < html >: The root element
  • < head >: Contains meta-information (e.g., title, character set, and links to CSS).
  • < body >: Contains the actual content of the webpage

Doctype

Question: What is Doctype? Why do you need it?

Answer: Doctype is an instruction to the browser to inform about the version of html document and how browser should render it. It ensures how element should be displayed on the page by most of the browser. And it also makes browser's life easier. otherwise, browser will guess and will go to quirks mode. Moreover, doctype is required to validate markup.

span vs div

Question: What is the difference between span and div?

Answer: div is a block element and span is inline element. It is illegal to put block element inside inline element. div can have a p tag and a p tag can have a span. However, span can't have a div or p tag inside.

div, section & article

Question: When should you use section, div or article?

Answer: section, group of content inside is related to a single theme, and should appear as an entry in an outline of the page. It’s a chunk of related content, like a subsection of a long article, a major part of the page (eg the news section on the homepage), or a page in a webapp’s tabbed interface. A section normally has a heading (title) and maybe a footer too.

article, represents a complete, or self-contained, composition in a document, page, application, or site and that is, in principle, independently distributable or reusable, e.g. in syndication. This could be a forum post, a magazine or newspaper article, a blog entry, a user-submitted comment, an interactive widget or gadget, or any other independent item of content.

div, on the other hand, does not convey any meaning, aside from any found in its class, lang and title attributes.

Authors are strongly encouraged to view the div element as an element of last resort, for when no other element is suitable. Use of more appropriate elements instead of the div element leads to better accessibility for readers and easier maintainability for authors.

Svg vs Canvas

Question: What are the difference between svg and canvas?

Answer: SVG (Scalable Vector Graphics) is a vector-based format where graphics are made up of shapes like lines and circles, making it ideal for scalable and interactive graphics that remain sharp at any size. In contrast, Canvas is a bitmap-based format that allows for drawing graphics pixel by pixel using JavaScript, suited for performance-heavy applications like games or animations, but its graphics are resolution-dependent and cannot be scaled without losing quality. SVG elements are part of the DOM and can be styled and interacted with individually, while Canvas is a single drawing surface, and its content cannot be directly manipulated once rendered.
